# Demo 与展示模式

## 首次使用与功能地图

- 首次打开非展示模式页面时，会显示一次 LedgerMind OS 使用引导。
- 首页“我现在该做什么”会根据交易、待确认、草稿、预算风险和数据质量生成真实优先建议。
- “快速开始”提供导入、审核、智能体和报告四步入口，可通过“我知道了”收起。
- 点击首页“功能地图”或设置页中的“功能地图”，可以按“导入账本 → 整理账本 → 分析财务 → 财务智能体 → 数据与隐私”理解全部核心模块。
- 展示模式不会强制弹出首次引导，避免遮挡 README 截图。

推荐体验顺序：加载演示账本 → 查看智能审核箱 → 向智能体提问 → 查看报告中心 → 导出 Markdown 月报。

安全边界：智能体只生成分析、任务和修改草稿，不会擅自修改账本；修改类动作必须由用户确认，并写入操作审计。

## 打开展示模式

可以点击 PC 顶部的“展示模式”，或直接访问：

```text
http://localhost:8787/?display=1
```

展示模式只改变页面呈现，不修改真实账本。用户名、商户和金额会使用演示内容，健康度、月报和智能体回答会显示完整示例。

## 加载演示账本

首次运行引导和顶部工具栏都提供“加载演示账本”入口。演示数据固定写入独立用户：

```text
ledgermind-demo
```

演示账本包含微信、支付宝、手工交易、收入、支出、退款、待确认、预算风险、周期扣费和中文规则，不会覆盖其他用户。

删除方式：加载后页面顶部会显示“演示账本”提示条，点击“清除演示账本”并确认。系统只删除 `ledgermind-demo`，随后返回进入演示前的用户，不会影响真实账本。

也可以直接访问：

```text
http://localhost:8787/?demo=1
```

## Showcase

访问：

```text
http://localhost:8787/showcase
```

该页面用于 GitHub 访客预览和产品展示，不会操作真实账本。

Showcase 中的“加载演示账本”会进入独立演示用户；“进入演示模式”只启用脱敏呈现。

## Agent OS 演示隔离

- 待执行草稿箱、操作审计和导出记录使用展示模式专属命名空间。
- 演示账本使用独立用户 `ledgermind-demo`，不会覆盖真实用户账本。
- 数据质量修复建议和报告中心使用匿名示例数据。
- 展示模式中的确认动作不会修改真实账本。
