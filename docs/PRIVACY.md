# 隐私与数据安全

LedgerMind OS 处理的账单、商户、金额和分类信息属于敏感个人数据。项目采用本地优先设计，但最终的数据安全责任仍由部署者和使用者承担。

## 智能体安全边界

- 智能体不会直接修改账本，修改类建议先进入待执行草稿箱。
- 规则、预算、记忆、周期扣费和批量分类需要用户确认。
- 草稿默认 24 小时后过期，重新执行前必须再次确认。
- 操作审计不记录 API Key、完整账单文件或交易原文。
- 展示模式、演示草稿和演示日志与真实用户数据隔离。
- 云端模型失败时使用本地账本分析回退。
- 数据质量修复建议只使用当前账本统计，不上传完整账单。
- 报告中心的导出记录仅保存月份、时间和文件名。

## 本地优先

- PC 端默认将账本保存到本机 `data/ledger-store.json`。
- Android 端可以在不连接 PC 后端时使用本地离线账本。
- 本地数据文件和 `.env` 已加入忽略规则，不应提交到 Git 仓库。
- 移动端离线模式使用当前设备的本地存储，不要求 PC 后端持续运行。

## 模型与 API Key

- SiliconFlow / DeepSeek 为可选能力，不配置 API Key 时仍可使用本地分析。
- 不要把真实 API Key 写入源码、README、截图或公开 Issue。
- 建议通过 `.env` 或设备设置保存 Key，并定期轮换。
- 使用云端模型意味着相关问题和必要的账本摘要可能发送给对应服务商，请自行阅读其隐私条款。
- SiliconFlow / DeepSeek 调用失败、超时或未配置时，系统会回退到当前设备的本地账本分析。

## 展示模式

展示模式会将用户名替换为“演示用户”，将商户映射为示例商户，并使用示例金额与结构化回答。它适合 README 和 Issue 截图，但发布前仍应人工检查图片中是否包含输入框、系统通知或其他敏感信息。

## 备份与导出

- JSON 备份包含当前用户完整账本，请加密保存。
- CSV 导出可能包含商户、金额和交易时间，不要直接公开。
- 删除本地数据前建议先完成备份并验证文件可读。

## 清空数据

- “清空”只删除当前用户的数据，包括交易、导入记录、规则和预算。
- 演示账本固定使用 `ledgermind-demo`，页面提供专用“清除演示账本”按钮，不会删除其他用户的数据。
- 浏览器或 Android 系统层面的应用数据清除会移除本地离线账本，操作前应先导出 JSON 备份。

## 用户责任

使用者需要自行管理账本文件、备份、设备访问权限和 API Key。云端部署时应额外配置 HTTPS、身份认证、访问控制、日志脱敏和定期备份。
