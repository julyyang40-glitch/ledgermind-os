# 参与 LedgerMind OS 开发

感谢你愿意参与 LedgerMind OS。项目优先保持中文体验、本地隐私、账单兼容性和移动端离线能力。

## 开发环境

- Node.js 20 或更高版本
- npm
- 构建 Android 时需要 JDK 21 与 Android SDK

安装并启动 PC Web：

```bash
npm install
npm start
```

访问：<http://localhost:8787/>

## 提交 Issue

提交问题前请先搜索已有 Issue，并尽量提供：

- 操作系统、Node.js 版本和浏览器或 Android 版本
- 可复现的操作步骤、预期结果和实际结果
- 已脱敏的错误信息或展示模式截图
- 账单来源和文件格式，但不要上传真实账单、API Key 或其他敏感信息

功能建议请说明使用场景、预期收益，以及是否影响本地优先和离线能力。

## 提交 Pull Request

1. Fork 仓库并从最新主分支创建功能分支。
2. 保持改动聚焦，不混入无关重构或生成文件。
3. 新行为需要补充或更新测试。
4. 更新相关 README、docs 或 CHANGELOG。
5. 提交前运行 `npm test`。
6. 在 PR 中说明改动目的、验证方式、界面截图及兼容性影响。

请勿在 PR 中提交 `.env`、真实 API Key、真实账单、个人账本数据或本地构建缓存。

## 代码风格

- 沿用现有 TypeScript、原生 JavaScript 和 CSS 结构。
- 用户界面文案以自然、克制的中文为主。
- 金额统一使用人民币格式。
- 优先复用现有模块和数据结构，避免无必要的新依赖。
- 不破坏 PC Web、移动端离线、本地分析回退和展示模式。
- 对复杂逻辑添加简短注释，避免描述代码表面行为的冗余注释。

## 测试要求

基础回归：

```bash
npm test
```

涉及 Android UI 或离线逻辑时还应运行：

```bash
npm run android:apk
```

涉及账单解析时，请覆盖微信、支付宝、CSV、XLSX、编码识别、重复检测和待确认状态；测试数据必须匿名或虚构。

## 安全与隐私

发现可能泄露账本、Key 或设备数据的安全问题时，请不要在公开 Issue 中附带敏感样本。发布仓库地址确定后，应在安全策略中补充私密报告渠道。
